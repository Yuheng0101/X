# Changelog

本文档记录重构版的主要变更。旧项目历史记录保留在文末，便于追溯原始脚本演进。

## 2026-05-01

- 移除 `95598/query` 重写触发，服务模式统一以 `electricity/bill/all` 为入口。

- README 补充各通知渠道完整环境变量列表。

### 验证码策略

- 更新验证码策略，使用新的验证码绕过方式

### 后端路由混淆

- 更新网关名称

## 2026-04-19

### 结构与配置

- 接口命名对齐官方 Web 端命名。
- 缓存 key 继续沿用旧版 BoxJs 配置。

### 网络与调试

- 网络请求改为 axios 风格客户端，基于拦截器统一处理响应、异常日志和重试。
- 调试模式下输出请求头、请求体、状态码和响应体。
- 调试日志对敏感字段做遮挡处理。
- 打包时避免把 Node 专属通知依赖静态打进 iOS 代理工具脚本。

### 登录与业务逻辑

- 服务模式返回值改为兼容 `dompling/Script` 的国网小组件接口，返回每个户号的原始数据对象。
- 服务模式默认返回全部绑定户号。
- 近期用电修正为最近 5 天数据，不再只取单条记录。

### 通知与构建

- 新增 ESM 版 `sendNotify.mjs`，保留原作者和历史方案出处说明。
- Node.js / 青龙环境支持 Bark、Telegram、Server 酱、钉钉、企业微信、iGot、PushPlus、WxPusher、Gotify、go-cqhttp、Bncr 等通知渠道。
- Telegram 支持代理配置，兼顾中国大陆网络环境。
- iOS 代理工具继续使用原生通知，不受 Node 外部通知逻辑影响。

### 文档

- README 重写为使用说明，CHANGELOG 独立维护更新记录。
- `.env.example` 补充完整中文注释，并明确重试参数不对用户开放。
